# Implementation Plan: Consumption Insights & Analytics

**Branch**: `004-consumption-insights` | **Date**: 2026-03-26 | **Spec**: [spec.md](./spec.md)
**Input**: Feature specification from `/specs/004-consumption-insights/spec.md`

## Summary

Add a new "Insights" tab to the bottom navigation that displays a single scrollable page of consumption analytics. The page includes: summary stats (this-week/month counts, streak, top product), a bar chart of consumption trends with time range selector, a product breakdown donut chart with sortable table, a radar chart of flavour profile over time, and an expiry waste rate metric. All data is derived from existing tables — no schema changes required. Charts are rendered using Recharts (already installed).

## Technical Context

**Language/Version**: TypeScript 5.7 (strict mode)
**Primary Dependencies**: Next.js 15 (App Router), Convex, NextAuth v5, Tailwind CSS v4, shadcn/ui, Recharts (already installed)
**Storage**: Convex (sole backend) — existing `consumptionLogs`, `products`, `batches` tables
**Testing**: Vitest for unit tests
**Target Platform**: Mobile-first PWA, deployed to Vercel
**Project Type**: Web application (Next.js App Router)
**Performance Goals**: Insights page loads within 2 seconds; chart interactions within 1 second
**Constraints**: No new npm dependencies (Recharts already present); all data read-only from existing tables
**Scale/Scope**: Single user, up to 365+ consumption log entries per year
